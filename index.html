HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.

CSS

  • inline css: Inline styles apply to a specific HTML tag, using a style attribute with a CSS rule to style a particular page element only
  • inline style must be edited individually whenever you make a design change
  • internal style sheet: contains CSS rules for the webpage in the head section of the HTML file
  • the rules only apply to that page, and you can configure CSS classes and IDs, which are used to identify elements, to style multiple elements in the page code. Again, a single change to the CSS rule will apply to all tagged page elements
  • inline CSS and internal style sheets are totally acceptable methods under certain conditions, but the most common way to style a webpage is by using an external CSS style sheet
  • external css style sheet, is preferred in most cases because it allows developers to keep all their CSS rules in a seperate file. Makes design changes easier

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git pull origin main: to make sure that the current branch is up to date before opening a pull request
  • ls -al ~/.ssh: in terminal to see "if" theres any existing ssh keys
  • ssh-keygen -t ed25519 -C "your_email@example.com": to create a new ssh key
  • clip "left pointing arrow" ~/.ssh/id_ed25519.pub: to copy ssh key
  • git checkout main: to leave feature branch
